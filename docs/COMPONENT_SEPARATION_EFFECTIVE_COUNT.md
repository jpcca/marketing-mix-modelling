# Component Separation vs Effective Component Count

**Date:** 2026-04-06

## Purpose

This note documents the benchmark scatter plot proposed for mixture-recovery analysis.

The goal of the plot is to show that, as the true data-generating components become more separated, the fitted model recovers a larger effective number of distinct components.

For a synthetic or labeled dataset with true component labels, the desired qualitative behavior is:

- when the true components overlap heavily, the effective estimated component count should stay closer to `1`
- when the true components become well separated, the effective estimated component count should move toward the true component count
- for a `K=3` DGP, the ideal limit is that the vertical axis approaches `3` as the horizontal-axis separation score approaches `1`

## Proposed Plot

- Horizontal axis: true component separation, computed from labeled component distributions
- Vertical axis: effective number of distinct estimated components, computed from the fitted mixture

This note focuses on the vertical-axis definition, because that is where the main design revision happened.

## Horizontal Axis Context

For labeled synthetic data, let `P_i` denote the empirical distribution of observations generated by true component `i`.

A convenient separation score is the average pairwise total variation distance:

\[
S_{\mathrm{true}}
=
\frac{2}{K(K-1)}
\sum_{i<j}\operatorname{TV}(P_i, P_j),
\qquad
\operatorname{TV}(P_i, P_j)
=
\frac{1}{2}\int |P_i(x)-P_j(x)|\,dx
\]

This score lies in `[0, 1]` when `P_i` and `P_j` are probability distributions.

- `S_true = 0` means all true components are indistinguishable
- `S_true = 1` means all component pairs are perfectly separated

For binned empirical distributions, the same quantity can be approximated by

\[
\widehat{\operatorname{TV}}(P_i, P_j)
=
\frac{1}{2}\sum_{b=1}^{B} |\hat p_{ib} - \hat p_{jb}|
\]

where `\hat p_{ib}` is the empirical probability mass of component `i` in bin `b`.

## Original Horizontal-Axis Direction

The initial horizontal-axis idea was based on pairwise overlap terms of the form

\[
\frac{\int P_i(x)P_j(x)\,dx}{2 - \int P_i(x)P_j(x)\,dx}
\]

aggregated across component pairs.

The intended use was to construct a scalar that changes as true components move farther apart.

## Problems With The Original Horizontal Axis

The overlap-based form above was useful as a first intuition, but it has several issues for the current benchmark figure.

### 1. It is easier to interpret as overlap than separation

As component distributions become more similar, the overlap term tends to increase. That makes it awkward for a figure where the desired reading is "moving right means easier separation."

### 2. The `[0, 1]` interpretation is not automatic without extra normalization assumptions

For general continuous densities, the raw integral `\int P_i(x)P_j(x)\,dx` is not inherently a distance and is sensitive to the scale and sharpness of the densities. That makes it harder to interpret as a clean separation axis.

### 3. It is less standard than total variation for distributional separation

The benchmark needs an axis that can be described cleanly in the paper and interpreted consistently across synthetic settings. Total variation has a direct probabilistic interpretation and a built-in `[0, 1]` range.

### 4. It mixes "closeness" and "distance" in a way that complicates the plot narrative

The benchmark figure is supposed to show that larger true separation corresponds to larger recovered effective component count. A quantity that naturally behaves like overlap would need to be inverted or re-labeled anyway.

## Why Total Variation Was Adopted For The Horizontal Axis

The horizontal axis was therefore changed to average pairwise total variation distance.

That choice has four practical advantages:

- it lies in `[0, 1]` for probability distributions
- it increases as component distributions become more separated
- it has a standard interpretation as a distance between distributions
- it can be approximated directly from labeled empirical data by binning

This makes it a better fit for the benchmark goal of showing a monotone relationship between true component separation and recovered effective component count.

## Why The Horizontal Axis Does Not Use Mixture Weights

The benchmark goal is to isolate the relationship between true component separation and the number of distinct fitted components recovered by the model.

For that reason, the horizontal axis is defined as an unweighted average over true component pairs rather than a weight-weighted separation score.

The main reasons are:

### 1. The horizontal axis should represent geometry, not prevalence

If the axis is weight-weighted, then separation and component prevalence get mixed together. That makes it harder to interpret the x-axis as a statement about how far apart the true components are.

### 2. The vertical axis already depends on estimated weights

The adopted effective-count metric uses flattened estimated mixture weights. Adding mixture weights to the horizontal axis as well would make both axes respond strongly to prevalence, which would blur the intended story of the figure.

### 3. The labeled-data setting makes a geometry-first definition possible

Because the immediate target is synthetic or otherwise labeled data, the true component distributions `P_i` are available. That lets the benchmark use an oracle-style separation measure directly, without needing to proxy identification difficulty through prevalence.

### 4. Weighting can be added later as a secondary recoverability analysis

A weight-weighted separation score may still be useful for practical detection difficulty, especially in real-data settings where tiny components are hard to identify even if they are well separated. That is a separate question from the current benchmark figure and can be introduced later as a supplementary analysis.

## Scope Of The Current Design

Under the current design:

- the horizontal axis is an unweighted true-component separation score
- the vertical axis is a similarity-adjusted effective count of fitted components

This separation of roles is intentional. The x-axis measures how separated the true latent structure is, while the y-axis measures how many distinct components the fitted model effectively recovered.

## Original Vertical-Axis Proposal

The original proposal for the effective component count was

\[
N_k
=
K_{\max}
-
\sum_{i<j} |\hat\pi_i - \hat\pi_j|
\]

where `\hat\pi_i` denotes the estimated mixture weight of component `i`.

This proposal has several appealing edge-case properties:

- if one component has weight `1` and all others have weight `0`, then `N_k = 1`
- if `m` components each have weight `1/m` and the rest have weight `0`, then `N_k = m`
- if all `K_max` components have equal weight, then `N_k = K_max`

These properties make it a reasonable weight-evenness score.

## Problems With The Original Vertical Axis

The main issue is that the quantity above depends only on the estimated weights. It does not use any notion of component similarity or separation.

That leads to the following problems.

### 1. It measures weight dispersion, not distinct recovered components

If the model splits one substantive component into two nearly identical fitted components, the score increases even though the fitted mixture did not recover an additional distinct response pattern.

### 2. Different weight vectors can produce the same score

For `K_max = 4`, both of the following yield `N_k = 2`:

- `(0.5, 0.5, 0, 0)`
- `(2/3, 1/6, 1/6, 0)`

The first case looks like two active components. The second has three positive weights. The score cannot distinguish them.

### 3. It can overstate recovery when fitted components are redundant

If three fitted components have nontrivial weights but two of them are nearly identical Hill curves, the score still stays high. That is not aligned with the scientific question of whether the model recovered distinct latent response segments.

### 4. It is too sensitive to prevalence for the intended figure

The target figure is meant to show that greater true separation leads to recovery of more distinct fitted components. A pure weight-based score can stay low simply because one true component has a smaller mixture weight, even when all true components are clearly separated.

This is undesirable for the intended interpretation of the plot, especially when the vertical axis is allowed to be continuous but should still behave like an effective component count.

## Adopted Vertical Axis

We therefore adopt a similarity-adjusted effective component count.

Let:

- `\hat\pi_i` be the estimated mixture weight of fitted component `i`
- `0 < \gamma \le 1` be a weight-flattening exponent
- `a_i^{(\gamma)}` be the renormalized flattened weights
- `S_{ij} \in [0, 1]` be a fitted-component similarity score, with `S_{ii} = 1`

Define the flattened weights by

\[
a_i^{(\gamma)}
=
\frac{\hat\pi_i^{\gamma}}{\sum_{\ell=1}^{K_{\max}} \hat\pi_\ell^{\gamma}}
\]

and define the vertical axis as

\[
N_{\mathrm{eff}}^{(\gamma)}
=
\frac{1}{(a^{(\gamma)})^\top S a^{(\gamma)}}
=
\frac{1}{
\sum_{i=1}^{K_{\max}}\sum_{j=1}^{K_{\max}}
a_i^{(\gamma)} a_j^{(\gamma)} S_{ij}
}
\]

This is the similarity-adjusted analogue of an inverse-Simpson effective count.

## Why This Definition Was Chosen

This form was preferred over the unadjusted version

\[
\frac{1}{\hat\pi^\top S \hat\pi}
\]

because the figure is intended to reflect the number of distinct recovered components, not just the prevalence-weighted diversity of the fitted mixture.

Using `\gamma < 1` reduces the tendency for one dominant component to suppress the count too aggressively. In other words:

- `\gamma = 1` emphasizes prevalence
- `\gamma < 1` gives more credit to smaller but clearly distinct fitted components

That tradeoff is appropriate for the benchmark figure, where the main question is whether distinct components become recoverable as the true DGP separation increases.

## Interpretation And Properties

The adopted score has the following properties when `S_{ii} = 1` and `0 \le S_{ij} \le 1`:

### 1. It is continuous

Small changes in weights or pairwise similarity lead to small changes in the score.

### 2. It discounts redundant components

If two fitted components are nearly identical, then `S_{ij}` is close to `1`, which prevents them from being counted as two fully distinct components.

### 3. It rewards separated components

If fitted components are well separated, then `S_{ij}` is close to `0` for `i \neq j`, and the effective count increases.

### 4. It reduces to a standard effective-count formula under perfect separation

If all fitted components are mutually separated, then

\[
N_{\mathrm{eff}}^{(\gamma)}
=
\frac{1}{\sum_i (a_i^{(\gamma)})^2}
\]

which is the inverse-Simpson count of the flattened weights.

### 5. It reaches the intuitive integer target in the ideal equal-weight case

If `m` fitted components are equally weighted and mutually separated, while the rest have zero weight, then

\[
N_{\mathrm{eff}}^{(\gamma)} = m
\]

In particular, a perfectly recovered `K=3` fit with three separated and similarly weighted components yields a value near `3`.

## Similarity Matrix Design

The design only requires a bounded similarity score with

\[
S_{ii} = 1, \qquad 0 \le S_{ij} \le 1
\]

One practical option is to define a distance `D_{ij} \in [0, 1]` between fitted component distributions and then set

\[
S_{ij} = 1 - D_{ij}
\]

For example, if `\hat P_i` and `\hat P_j` denote fitted component distributions over a chosen observable quantity, one can use

\[
D_{ij} = \operatorname{TV}(\hat P_i, \hat P_j)
\]

or a binned empirical approximation of total variation distance.

The exact observable used to define `\hat P_i` still needs to be fixed in implementation. For the current benchmark note, the key point is that the vertical axis must use a fitted-component similarity term rather than weights alone.

## Numerical Stability

The adopted definition does not suffer from divide-by-zero when implemented correctly.

Because the flattened weights satisfy `a_i^{(\gamma)} \ge 0` and `\sum_i a_i^{(\gamma)} = 1`, and because `S_{ii} = 1`, the denominator is bounded below by

\[
(a^{(\gamma)})^\top S a^{(\gamma)}
\ge
\sum_i (a_i^{(\gamma)})^2
>
0
\]

Therefore:

- complete separation does not produce division by zero
- the denominator only vanishes if the similarity matrix is mis-specified

In code, it is still reasonable to guard the denominator with a small `eps` for numerical safety.

## Recommended Initial Hyperparameter Choice

The main open hyperparameter is `\gamma`.

Recommended starting point:

- default trial value: `\gamma = 0.5`

This choice keeps the metric continuous, reduces excessive dominance by the largest fitted component, and still preserves monotonic sensitivity to the fitted weight distribution.

Sensitivity checks over values such as `0.3`, `0.5`, and `1.0` are recommended before the metric is locked into the benchmark reporting layer.
